# План: Интеграция инсайтов коллеги в Sreda на базе референса EDP

## Референс: edp-med.vercel.app (РЖД-Медицина EDP)

### Что забираем из референса

| Паттерн EDP | Как адаптируем для Sreda | Где в Sreda |
|-------------|--------------------------|-------------|
| **Список сотрудников** — таблица с аватаром, ФИО, специализацией, филиалом, рейтингом | **Штат агентов** — список агентов с аватаром, именем, ролью, отделом, KPI, статусом | Новый экран: «Штат агентов» в Менеджменте |
| **Детальная карточка** — аватар, бейджи должности, ID, рейтинг, двухуровневые вкладки | **Профиль агента** — аватар, бейджи (роль, версия), ID агента, рейтинг, вкладки: Общее / KPI / Бюджет / SLA / Доступы / Логи | Новый экран: клик по агенту в штате |
| **KPI-карточки** — 6 метрик в сетке, каждая с источником данных (МИС) | **KPI агента** — задачи выполнено, ошибки, экономия времени, стоимость, качество, с источником (аудит, аналитика) | Вкладка KPI в профиле агента |
| **Интеграционные бейджи** — МИС, 1С, Личный кабинет, КЗД, К-TEAM | **Системные интеграции агента** — CRM, Биллинг, Репозиторий, Документы, HR — с статусом подключения | В профиле агента, блок «Интеграции» |
| **Данные с пометками источников** — (1С), (КГ) | **Данные с provenance** — каждое поле в профиле агента помечено источником: ассистент, аудит, ручной ввод | Во всех карточках агента |
| **Мульти-филиальность** — список клиник в разных городах | **Мульти-филиальность** — список филиалов/дочек компании, переключатель в хедере | Новый экран: «Филиалы» в Менеджменте |
| **Рейтинг звёздами + отзывы** | **Рейтинг агента** — звёзды + количество отзывов/оценок от пользователей | В списке агентов и в профиле |
| **Бонусные баллы** | **Бюджет агента** — остаток бюджета, burn rate, лимиты | Вкладка «Бюджет» в профиле агента |

## Этап 1. Штат агентов (Agent Workforce)

**Цель:** агент перестаёт быть «инструментом» и становится «цифровым сотрудником» с профилем, KPI, бюджетом.

### Новые сущности в mock.js

```js
const AGENT_WORKFORCE = [
  { id:'a-001', name:'Копирайтер-α', role:'copy', dept:'marketing', 
    avatar:'🤖', version:'v2.3', status:'active', 
    kpi:{ tasksDone: 142, errors: 2, timeSaved: 340, cost: 12400, quality: 4.6 },
    budget:{ daily: 500, spent: 340, currency:'₽' },
    sla:{ responseTime:'<3s', uptime:'99.7%', accuracy:'96%' },
    integrations:['CRM','Drive','Mail'], 
    source:'Среда · Студия', hired:'2025-03-12' },
  // ... ещё 15-20 агентов
];
```

### Новый экран: «Штат агентов» (в Менеджменте)

**Верхняя панель:**
- Заголовок: «Цифровой штат · N агентов в M отделах»
- Фильтры: по отделу, по роли, по статусу, поиск по имени
- Переключатель: «Агенты / Люди» (чтобы видеть смешанную команду)

**Таблица (как в EDP):**
- Колонки: ID | Аватар | Имя | Роль | Отдел | KPI | Бюджет/день | Статус | Рейтинг
- Сортировка по любой колонке
- Клик по строке → профиль агента

**Карточки KPI в шапке (как в EDP):**
- Всего агентов · N
- Активных сегодня · M
- Задач выполнено · K
- Экономия часов · H
- Burn rate · ₽X/день

## Этап 2. Профиль агента (Agent Profile)

**Цель:** полная карточка «цифрового сотрудника», аналогичная карточке врача в EDP.

### Структура (как в EDP — двухуровневые вкладки)

**Верхний блок (как в EDP):**
- Аватар агента (круглый, цветной фон)
- Имя: «Копирайтер-α»
- Должность: «Копирайтер · Маркетинг»
- Бейджи: «Копирайтер» (цветной), «v2.3» (версия), «a-001» (ID)
- Рейтинг: 4.6 ★★★★★ (34 оценки)
- Иконка «журнал» → лог действий

**Интеграционные бейджи (как МИС/1С в EDP):**
- CRM (зелёный — подключено)
- Google Drive (зелёный)
- Почта (зелёный)
- Биллинг (жёлтый — ограниченный доступ)
- Репозиторий (красный — нет доступа)

**Вкладки верхнего уровня (как Личные/Трудовые/Образование в EDP):**
1. **Общее** — описание, зона ответственности, навыки, версии
2. **KPI** — метрики в карточках
3. **Бюджет** — лимиты, burn rate, прогноз
4. **SLA** — соглашения, uptime, время отклика
5. **Доступы** — RBAC, какие данные видит
6. **Логи** — аудит действий агента

**Под-вкладки KPI (как Вознаграждение/KPI/Клинический профиль в EDP):**
- Производительность
- Качество
- Стоимость
- Удовлетворённость

**Карточки KPI (как в EDP):**
```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│   142       │ │    2        │ │   340 ч     │ │   ₽12 400   │
│   шт.       │ │   шт.       │ │             │ │             │
│ Задач       │ │  Ошибок     │ │ Экономия    │ │  Стоимость  │
│  (аудит)    │ │  (аудит)    │ │  времени    │ │   (фин)     │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

## Этап 3. Канал отдела (Department Channel)

**Цель:** создать «Slack-подобное» пространство для коллаборации людей и агентов внутри отдела.

### Новый экран: «Канал отдела» (в кабинете каждой роли)

**Структура (как чат + лента):**
- Левая колонка: список «тем»/«потоков» в отделе
  - #общий
  - #задачи
  - #kpi
  - #вопросы-агентам
  - #передачи
- Центральная колонка: лента сообщений
  - Сообщения от людей (с аватаром)
  - Сообщения от агентов (с аватаром 🤖, цветной рамкой)
  - Системные события (передачи, апрувы, KPI)
- Правая колонка: участники канала (люди + агенты)

**Типы сообщений:**
- Человек: «Нужен черновик лендинга к четвергу»
- Агент: «Готов черновик. 3 секции, CTA. На проверку.» + карточка черновика
- Система: «Оля передала КП «Гамма» → Ира (Юр)»
- Агент-репорт: «KPI за сегодня: 12 задач, 0 ошибок, ₽340»

**Ввод сообщения:**
- Composer с упоминаниями (@агент, @человек)
- Агенты отвечают в тред

## Этап 4. Масштаб холдинга (Multi-Tenant)

**Цель:** подготовить Sreda к масштабу «200 предприятий, 100 систем».

### Новые сущности

```js
const FILIALS = [
  { id:'f-001', name:'Среда · Москва Центральная', city:'Москва', status:'active', people:52, agents:28 },
  { id:'f-002', name:'Среда · СПб Главная', city:'Санкт-Петербург', status:'active', people:34, agents:19 },
  // ...
];
```

### Изменения в UI

**Переключатель филиала в топбаре:**
- Рядом с «Поставить задачу среде» — дропдаун «Москва Центральная ▾»
- При переключении — меняются данные, агенты, команда

**Экран «Филиалы» (в Менеджменте):**
- Список, как «Клиники» в EDP
- Колонки: ID | Название | Город | Людей | Агентов | Статус
- Клик → дашборд филиала

**Пульс компании → Пульс холдинга:**
- Сводка по всем филиалам
- KPI холдинга = сумма KPI филиалов

## Этап 5. Позиционирование vs конкурентов

**Цель:** явно отделить Sreda от Cursor/Devin.

### Новый экран: «Battle · позиционирование» (в Владелец платформы)

**Матрица сравнения:**
| Критерий | Sreda | Cursor Enterprise | Claude Code | Devin Teams |
|----------|-------|-------------------|-------------|-------------|
| Фокус | 8 ролей компании | Разработка | Разработка | Разработка |
| Агенты | Штат с KPI/бюджетом | IDE-ассистент | IDE-ассистент | Автономный dev |
| Передачи | Сквозные потоки | PR review | PR review | Нет |
| Governance | Сениорити + конституция | Нет | Нет | Нет |
| Масштаб | Холдинг | Команда | Команда | Команда |

## Порядок реализации

1. **Штат агентов + профиль** — фундамент, даёт концепцию «агент как сотрудник»
2. **KPI и бюджет** — метрики в профиле
3. **Канал отдела** — коллаборация
4. **Масштаб холдинга** — филиалы
5. **Battle позиционирование** — маркетинг
