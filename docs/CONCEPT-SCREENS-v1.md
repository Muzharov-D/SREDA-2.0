# СРЕДА — Спецификация экранов v1

> Пять экранов новой модели. Каждый — проекция «Концепции платформы СРЕДА в5» и документа [concept-interface-v1.md](../concept-interface-v1.md).
> Правило вывода: `экран = f(роль, контексты, ЦС, состояние задач)`. Ничего не рисуется в обход модели.
> Единая грамматика всех экранов:
> - **Подсветка — два цвета:** 🔴 «блок на тебе» (приёмка / санкция), 🟡 «вопрос к тебе» (уточнение / согласование). Всё, что ЦС делает сам — без подсветки.
> - **Исполнитель всегда подписан:** человек или ЦС.
> - **Провенанс:** у кандидата/знания виден источник.

---

## Экран 1. Кокпит РЦС — Пульс «Я»

**Кто:** любой пользователь (как РЦС своих ЦС). **Высота:** личный контекст (§4.1).

**Топбар:** логотип (→ Пульс) · переключатель высоты `[ Я | Отдел | Компания ]` (Компания — замок, только Оркестратор) · метр ₽ ИИ (агент учёта §5.3) · аватар роли.

**Левый рельс «Мой штат»:** список ЦС/людей, где я РЦС; статус-точка (в потоке / ждёт / простой); `+ штат` → Конструктор (Экран 2).

**Центр «Пульс»** (одна колонка, сверху вниз по приоритету):
1. **Ждёт меня** — точки участия §7.2: `приёмка` (🔴), `уточнение` (🟡), `санкция` (🔴). С инлайн-действиями (Принять / Ответить / Разрешить).
2. **Встречи дня** — с кнопкой «бриф от помощника» (§5.2).
3. **Мои ЦС в работе** — прогресс, без подсветки (ЦС сам).
4. **Предложено помощником** — кандидаты из почты/Zoom (Экран 3), идеи оптимизации.

**Низ:** личный помощник — сквозная строка, знает текущий экран (§5.2), ⌘K.

**Состояния:**
- Пустой Пульс = «Всё под контролем. N ЦС в работе, ближайшее решение в HH:MM» (спокойствие как фича, не пустой экран).
- Смена высоты меняет рельс и контекст помощника.

**Данные:** мои ЦС (где РЦС), задачи в статусах §7.2, встречи (СК/календарь), метр — агент учёта.

## Экран 2. Конструктор + эскалация

**Кто:** пользователь на своём рабочем месте. **Вход:** `+ штат` / пустоты Пульса.

**Не «холст с виджетами»**, а один вопрос **«чего не хватает?»** с двумя исходами:
- **Есть в системе** (блок / ЦС / контекст) → self-service, добавить мгновенно (каталог слева).
- **Нет в системе** → **эскалация ЦС-администратору платформы**, который провижинит инструмент (MCP) / навык / нового ЦС (§8). Карточка статуса «в работе».

**Жизненный цикл (§4.3):** личное (мой экран, черновик) → контекст отдела → компания = новый **дефолт роли для всех**. Подъём — только с санкцией владельца контекста / Оркестратора, с провенансом.

**Продуктовое требование:** дефолт роли обязан быть так хорош, чтобы ~80% не открывали конструктор. Метрика успеха — **как редко в него заходят**. Каждая эскалация = телеметрия спроса + топливо библиотеки (§9).

**Данные:** каталог доступных блоков/ЦС/контекстов; очередь эскалаций к админ-ЦС.

## Экран 3. Кандидаты из Zoom / почты → задачи

**Кто:** пользователь. **Вход:** «Предложено помощником» в Пульсе, либо открытие транскрипта/письма.

**Принцип:** ничего не создаётся молча. Транскрипт/протокол (§3.2) → **кандидаты**, человек правит и подтверждает → только тогда постановка ЦС (§7.2).

**Каждый кандидат:** чекбокс (подтвердить/снять) · формулировка · **исполнитель** (ЦС/человек) · срок · **источник-цитата** из звонка (провенанс).

**Умный дедуп:** помощник сверяет с памятью контекста (§4.2) и снимает то, что уже сделано («презентацию уже отправляли»).

**Финал:** «Поставить отмеченные · N задач + M встреч». Кнопка «Открыть транскрипт».

**Данные:** протокол встречи (инструмент протоколирования), память контекста, штат для назначения.

## Экран 4. Пульс руководителя — «Отдел» / «Компания»

**Кто:** «Отдел» — пользователь по своим контекстам; «Компания» — только Оркестратор (§6). **Та же модель, зум наружу.**

**Отдел:**
- **Поток передач** (§7.2): узел = реальный ЦС/человек, синапс = реальная передача. Застрявший **гейт** подсвечен 🔴 (импульс не идёт дальше — ждёт санкции/согласования).
- **Ждёт отдел** — точки участия уровня отдела.
- **Загрузка штата** — люди + ЦС, % занятости (перегруз ЦС — жёлтым).

**Компания (Оркестратор):** тот же вид на всю организацию — узлы = отделы, синапсы = межотделовые передачи, висящее на стыках.

> Переосмысляет старую «нервную систему» из [concept-redesign.md](../concept-redesign.md): метафора выжила, но теперь каждый импульс привязан к данным (узел = реальный ЦС, синапс = реальная передача, замок = конкретный гейт §7.2).

**Данные:** FLOWS/передачи, board (статусы), HEADCOUNT/DIGITAL_HEADCOUNT (загрузка).

## Экран 5. Онбординг компании (штатное расписание)

**Кто:** Администратор / Оркестратор + его ассистент. **Не для рядового пользователя.**

**Поток:** ТЗ на входе → ассистент раскладывает на **оргструктуру** (отделы + роли) → по каждой роли подбирает **ЦС из библиотеки** (`взять`) или, если нет, предлагает **создать** (`создать ЦС`) → назначение РЦС → «Укомплектовать компанию».

**Дифференциация (§9):** компания не *настраивается*, а *укомплектовывается штатом*. Каждый «создать ЦС» обогащает **библиотеку/маркетплейс ролей** — сетевой эффект.

**Данные:** ТЗ, библиотека ЦС (renderLibrary/marketplace), роли (§6), оргструктура.
