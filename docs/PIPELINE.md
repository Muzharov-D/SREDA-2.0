# Пайплайн разработки Авандок Среда — из существующих активов в продукт

> Синтез пяти документов-гипотез (бриф v5 «Цифровой EPAM», архитектура КОРУС v1.0,
> обзор и презентация для руководителей, HTML-архитектура v2) + UX-прототип Авандок Среда 2.0.
> Дата: июнь 2026.

## 0. Что мы хотим (сведённая формула)

**Одна платформа, одна история, три слоя выручки.**

Продукт: **платформа цифровых сотрудников** — ИИ-агентов, которых бизнес нанимает
и которыми управляет как людьми (резюме → собеседование → ДИ → доступы через ИБ → KPI),
по цене ~1/3 ФОТ, с маржой ≥30%, on-premise или SaaS.

Ведущая история для рынка РФ (2026–2027): **«цифровые сотрудники, которым можно доверять»** —
экономика, суверенитет данных, управляемость. История «цифровой EPAM / продажа
агенто-результата на рынке $638B» — это глобальный этап-2, когда маркетплейс и
data flywheel наберут массу. В питче они идут как видение, не как обещание на завтра.

## 3. Фазы (синхронизированы с роадмапом архсовета)

### Фаза 0 — Решения (2 недели)
- Архсовет: утвердить 4 решения (Agent Runtime как сервис; SPIFFE+PEP; gVisor/Firecracker; trace-context).
- Свести бренд: одно имя (Авандок Среда / СРЕДА) и один дек наружу.
- Зафиксировать политику обучения: что обобщается глобально (навыки), что остаётся в тенанте (данные, память).
- Выбрать якорного заказчика пилота и 2 роли.

### Фаза 1 — Энтерпрайз-фундамент (0–3 мес)
Воркстримы:
- **Платформа**: Agent Runtime → отдельный сервис; SQLite → PostgreSQL; Helm; sandbox gVisor.
- **Gateway**: LiteLLM с тегированием project/task/agent_id (OpenTelemetry); BYOK; fallback v1.
- **ИБ**: OpenBao; ABAC (OPA) поверх RBAC; вынос секретов из SQLite.
- **Cowork v0 (наш прототип → код)**: namespace /cowork в Sreda; Пульс (реальные события ранов), реестр агентов, ДИ-редактор. Прототип = дизайн-спека, перенос 1:1.
- **Пилот**: 2 роли у якорного заказчика; KPI: ₽/задача, % принятых без правок, экономия часов.
Выход: пилотный отчёт с доказанной экономикой → материал для продаж.

### Фаза 2 — Знания и Cowork GA (3–6 мес)
- Реестры Skills/MCP: подпись, SBOM-скан, песочница, аппрув ИБ.
- RAG (Qdrant) с партиционированием тенант×допуск; Wiki (Outline).
- Локальные модели через LLMaaS Впрод в контуре первого on-prem заказчика.
- Cowork v1: золотые профили, бюджеты ИИ, аудит-UI, передачи.
- Метеринг OpenMeter/Lago + BI (Metabase); прайс-лист по ролям.
- **Eval-harness** (критический IP): бенчмарки по ролям, метрики cost/задача, time-to-accept, доля возвратов.

### Фаза 3 — Масштаб и аттестация (6–12 мес)
- K8s Data Plane + Firecracker; GPU-пул, scale-to-zero.
- SPIFFE/SPIRE + синхронный PEP на каждом действии; SIEM (Wazuh).
- Air-gapped; опции СКЗИ (КриптоПро) для ПДн/КИИ.
- Agents Portal: публичная витрина, собеседования с тестовыми заданиями, маркетплейс сторонних агентов (комиссия).
- HA Gateway, multi-region SaaS.

## 4. Команда минимум (фаза 1)
- Продакт-владелец (один на всё, сводит бренд и приоритеты)
- 2 backend (Runtime, Gateway, метеринг) · 1 frontend (Cowork из прототипа)
- 1 DevOps/платформа (K8s, Helm, sandbox) · 1 ИБ-инженер (OpenBao, ABAC, PEP-дизайн)
- 0.5 аналитик качества (eval-harness с фазы 1, не «потом»)

## 5. Гейты между фазами
- Ф0→Ф1: 4 решения утверждены, бренд один, пилотный заказчик подписан.
- Ф1→Ф2: пилот показал экономику (≥30% маржа на задачах пилота), trace-context работает сквозь все слои.
- Ф2→Ф3: 3 платящих тенанта, eval-метрики стабильны, 0 инцидентов межтенантной утечки.
