# Дорожная карта

## Phase 1: Foundation — заложить архитектурный фундамент (0–3 месяца)

Первый квартал развития Авандок.Среды посвящён созданию надёжного, безопасного и масштабируемого ядра платформы. Мы строим не просто обёртку над LLM, а полноценный **Agent Runtime** — среду исполнения интеллектуальных агентов с поддержкой мульти-языковой экосистемы (Node.js и Python), встроенным reasoning-loop для многошаговой логики, расширяемым набором инструментов (tools) и обязательным механизмом human-in-the-loop для критически важных операций. Этот runtime станет сердцем всей платформы, обеспечивая предсказуемое поведение агентов и полный контроль над их действиями.

Параллельно разворачивается **LLM Gateway** на базе LiteLLM с интеграцией OpenRouter, что даёт заказчикам единый интерфейс к десяткам языковых моделей без vendor lock-in. Gateway реализует trace-context для полной трассировки запросов, что критично для отладки, аудита и оптимизации затрат на токены. Каждый вызов модели становится прозрачным и измеримым.

Безопасность закладывается на уровне инфраструктуры: **OpenBao** (форк HashiCorp Vault под открытой лицензией) берёт на себя управление секретами, ключами API и сертификатами, а **RBAC через OPA (Open Policy Agent)** обеспечивает детальное разграничение доступа с декларативными политиками. Это позволяет строить zero-trust архитектуру с первого дня.

Для облачного развёртывания готовится **Helm-чарт** с изолированной песочницей на базе **gVisor** — контейнерного sandbox-рантайма от Google, который добавляет дополнительный уровень изоляции между агентами и хост-системой. Это особенно важно при выполнении произвольного кода агентами.

Слой персистентности эволюционирует от **SQLite** на ранних этапах прототипирования к полноценной **PostgreSQL** для production-нагрузок, с поддержкой миграций, репликации и резервного копирования.

**Майлстоун: релиз Sreda v0.9 (private beta)** — закрытое бета-тестирование с ключевыми партнёрами, валидация архитектурных решений и сбор обратной связи по API и UX.

## Phase 2: Scale — масштабирование и формирование экосистемы (3–6 месяцев)

Второй этап фокусируется на превращении платформы из инфраструктурного ядра в полноценную экосистему для разработки и эксплуатации ИИ-агентов. Центральным элементом становится **Skills/MCP Marketplace** — каталог переиспользуемых навыков и MCP-серверов (Model Context Protocol) с обязательным **SBOM-сканированием** (Software Bill of Materials) каждого компонента. Это гарантирует, что сторонние интеграции не содержат уязвимостей и лицензионных конфликтов, что критично для enterprise-заказчиков.

Разворачивается полноценная **RAG-инфраструктура** на базе векторной СУБД **Qdrant** с поддержкой гибридного поиска (dense + sparse vectors), ре-ranking и индексации корпоративных документов и вики-систем. Агенты получают доступ к актуальной внутренней knowledge base компании с точным цитированием источников и контролем галлюцинаций.

Для снижения зависимости от проприетарных API и оптимизации TCO запускается хостинг open-weight моделей: **Qwen** и **DeepSeek** разворачиваются через **vLLM** — высокопроизводительный inference-движок с поддержкой PagedAttention, continuous batching и tensor parallelism. Это позволяет предлагать заказчикам приватный inference на собственной инфраструктуре без утечки данных за периметр.

Появляется **Cowork-UI MVP** — специализированный интерфейс совместной работы для Forge-проектов, где команды разработчиков, аналитиков и бизнес-пользователей взаимодействуют с агентами в едином рабочем пространстве с версионированием, code review и совместным редактированием.

Монетизация и аналитика получают инструментальную базу: **Metering Engine** на базе OpenMeter или Lago обеспечивает точный учёт потребления ресурсов (токены, вычислительные минуты, storage) с поддержкой сложных тарифных моделей, а **BI-дашборды** дают заказчикам прозрачную картину использования платформы и ROI от внедрения агентов.

**Майлстоун: релиз Sreda v1.0 (public launch, 10 июня 2026)** — публичный запуск платформы, открытая регистрация, первые платные клиенты, целевой показатель — стабильная нагрузка от 10+ активных команд.

## Phase 3: Enterprise — enterprise-grade надёжность и глобальное масштабирование (6–12 месяцев)

Финальный этап трансформирует Авандок.Среду в платформу, готовую к требованиям крупнейших корпораций и регулируемых отраслей. Инфраструктура вычислений переходит на **K8s Data Plane** с **Firecracker microVMs** — лёгкими виртуальными машинами от AWS, обеспечивающими изоляцию уровня VM для каждого агента с временем запуска менее 125 мс. Добавляются **GPU-пулы** для inference и fine-tuning на собственном железе, а механизм **scale-to-zero** автоматически останавливает неактивные воркеры, минимизируя инфраструктурные затраты.

Безопасность достигает уровня banking-grade: **SPIFFE/SPIRE** обеспечивает криптографическую идентификацию каждого сервиса и агента в mesh-сети, а **PEP (Policy Enforcement Point)** проверяет авторизацию каждого действия агента в реальном времени. Это создаёт непрерывный контур доверия от идентификации до исполнения.

Интеграция с **SIEM-экосистемой** (Wazuh для endpoint detection, Loki/ELK для централизованного логирования) позволяет корпоративным SOC-командам мониторить активность агентов в привычных инструментах, настраивать корреляционные правила и реагировать на инциденты по стандартным playbooks.

Для государственных заказчиков, оборонки и критической инфраструктуры реализуется опция **air-gapped deployment** — полностью автономная установка платформы без внешних сетевых соединений, с локальными репозиториями моделей и офлайн-активацией.

SaaS-инфраструктура достигает уровня high availability: **HA LLM Gateway** с автоматическим failover между провайдерами, **multi-region развёртывание** для compliance с требованиями data residency (GDPR, 152-ФЗ), гео-распределённое кэширование и репликация векторных индексов.

**Майлстоун: релиз Sreda v2.0 (enterprise-ready)** — сертификация SOC 2 Type II, поддержка FedRAMP/ФСТЭК-совместимых конфигураций, SLA 99.95%, выход на крупных enterprise-заказчиков и международные рынки.

Три фазы дорожной карты формируют последовательную, риск-ориентированную стратегию вывода продукта на рынок: от технологического фундамента через product-market fit к enterprise-доминированию. Каждый этап приносит измеримую ценность заказчикам и укрепляет конкурентные барьеры Авандок.Среды.
